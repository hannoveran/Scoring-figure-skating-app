# Суддівська платформа для одиночного фігурного катання

Веб-застосунок для автоматизації процесу оцінювання виступів спортсменів у змаганнях з одиночного фігурного катання.
Система дозволяє суддям швидко вводити оцінки, а також автоматично обчислює підсумкові результати виступів.

## Technologies

Проєкт реалізований із використанням сучасних технологій веб-розробки:

React – бібліотека для створення користувацьких інтерфейсів

React Router – навігація між сторінками застосунку

JavaScript (ES6+) – основна мова програмування

CSS3 – стилізація інтерфейсу

HTML5 – структура сторінок

Git & GitHub – контроль версій та зберігання коду

# Аналіз предметної області

## Актуальність теми

Фігурне катання є складним технічно-естетичним видом спорту, у якому оцінювання виступів спортсменів здійснюється за багатьма параметрами та потребує високої точності й оперативності. Традиційні способи фіксації оцінок, зокрема паперові протоколи або електронні таблиці, можуть ускладнювати підрахунок результатів і підвищувати ймовірність помилок.

У зв’язку з розвитком інформаційних технологій актуальним є створення програмних засобів, що автоматизують введення, обробку та відображення суддівських оцінок. Розробка веб-застосунку для суддівської платформи у одиночному фігурному катанні дозволяє спростити роботу суддів, підвищити точність розрахунків і прискорити отримання результатів.

## Об’єкт і предмет дослідження

**Об’єкт дослідження** – процес оцінювання виступів спортсменів у одиночному фігурному катанні.

**Предмет дослідження** – методи та програмні засоби реалізації веб-застосунку для введення, обробки та відображення суддівських оцінок під час змагань з одиночного фігурного катання.

---

## Практичне значення роботи

Практичне значення роботи полягає у створенні прототипу веб-застосунку, який може використовуватися як інструмент для підтримки процесу суддівства у змаганнях з одиночного фігурного катання.

Розроблений застосунок дозволяє:

- забезпечити зручне введення суддівських оцінок;
- автоматизувати обробку та підрахунок результатів;
- зменшити ймовірність помилок при ручному підрахунку;
- підвищити швидкість отримання підсумкових результатів;
- забезпечити зрозумілий та зручний інтерфейс для користувачів.

# Моделювання системи

## Актори системи

**Суддя** – основний користувач системи, який вводить оцінки за виконані елементи та компоненти програми спортсмена.

**Адміністратор** – користувач, який керує даними системи: створює змагання, додає спортсменів, налаштовує список елементів та контролює доступ до системи.

---

## Бізнес-логіка системи

Бізнес-логіка системи полягає в автоматизації процесу оцінювання виступів спортсменів під час змагань з одиночного фігурного катання.

Основні процеси роботи системи:

1. Адміністратор створює змагання та додає інформацію про спортсменів.
2. Суддя відкриває сторінку конкретного виступу.
3. Суддя вводить оцінки за технічні елементи та компоненти програми.
4. Система зберігає введені дані та автоматично обчислює підсумковий результат.
5. Після завершення виступу система формує підсумкові оцінки спортсмена та відображає результати.

---

## Функціональні вимоги (FR)

- **FR1** – система повинна дозволяти авторизацію користувачів.
- **FR2** – система повинна дозволяти створення та редагування змагань.
- **FR3** – система повинна дозволяти додавання та редагування даних спортсменів.
- **FR4** – система повинна забезпечувати введення суддівських оцінок.
- **FR5** – система повинна автоматично обчислювати підсумковий результат виступу.
- **FR6** – система повинна зберігати результати у базі даних.
- **FR7** – система повинна відображати результати виступів користувачам.

---

## Нефункціональні вимоги (NFR)

- **NFR1** – система повинна мати зручний та зрозумілий користувацький інтерфейс.
- **NFR2** – система повинна забезпечувати швидку обробку та відображення даних.
- **NFR3** – система повинна гарантувати коректне збереження інформації.
- **NFR4** – система повинна працювати у сучасних веб-браузерах.
- **NFR5** – система повинна забезпечувати базовий рівень безпеки доступу до даних.
